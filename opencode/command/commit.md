Analyze all changes since the last git commit and create small, atomic commits that follow the logical development path.
For each commit:

1. Group related changes by component/functionality
2. Use conventional commit format (feat/fix/docs/refactor/etc.)
3. Write clear commit messages explaining the "why"
4. Make commits as small as possible while keeping them functional
5. Follow the logical progression of implementing the feature

Before committing, show me the planned commits for approval.
